# Google Play store listing

Store listing text, release notes, and graphics for **Kids Zone**
(`dev.waybeyond.kidszone`), managed as code with
[`fastlane supply`](https://docs.fastlane.tools/actions/supply/) under
[`fastlane/`](../fastlane/).

> Builds are produced by EAS (see `eas.json` / the release workflow). Fastlane
> here manages **only** the Play Console store listing — it does not build or
> upload the app binary.
>
> `fastlane/README.md` is auto-generated by fastlane (a lane reference) and is
> overwritten on every run — keep human docs here instead.

## Layout

```
fastlane/
├── Appfile              # package name + path to service-account key
├── Fastfile            # lanes: metadata · changelog · validate · pull
└── metadata/android/
    ├── en-US/          # English (US) listing
    │   ├── title.txt
    │   ├── short_description.txt
    │   ├── full_description.txt
    │   ├── video.txt
    │   ├── changelogs/default.txt
    │   └── images/{icon,featureGraphic,phoneScreenshots,sevenInchScreenshots,tenInchScreenshots}
    └── ar/             # Arabic listing (full RTL)
        ├── title.txt
        ├── short_description.txt
        ├── full_description.txt
        ├── changelogs/default.txt
        └── images/{icon,featureGraphic,phoneScreenshots,tenInchScreenshots}
```

## Character limits (enforced by Play Console)

| File                     | Limit      |
| ------------------------ | ---------- |
| `title.txt`              | 30 chars   |
| `short_description.txt`  | 80 chars   |
| `full_description.txt`   | 4000 chars |
| `changelogs/*.txt`       | 500 chars  |

## Release notes / changelogs

`changelogs/default.txt` is the fallback fastlane uses for any version code.
Because EAS auto-increments the Android `versionCode` remotely
(`appVersionSource: "remote"`), we don't know the integer ahead of time — so the
listing keeps a single `default.txt` per locale. To pin notes to a specific
build, add `changelogs/<versionCode>.txt`.

### Everything targets an existing release

fastlane `supply` uploads listing text, images **and** changelogs inside one
per-version-code loop (`supply/uploader.rb#perform_upload_meta`). With no binary
to derive a version code from, it fails with
`Could not find release for version code '' to update changelog`.

So **every listing lane requires `version_code:` + `track:`** pointing at an
existing release — not just changelogs. Run `fastlane tracks` to list them:

```
$ fastlane tracks
track="production": (no releases)
track="alpha": [completed] vc=4 name="v1.0.1"
track="internal": [draft] | [completed] vc=4 name="v1.0.1"
```

The closed-testing release lives on the **`alpha`** track (`production` is empty),
so the working invocation is `track:alpha version_code:4`.

## Images

Current set (already accepted by Google for closed testing — do not "fix" to
match nominal specs):

- `icon/` — 512×512 PNG (alpha allowed for the hi-res icon)
- `featureGraphic/` — 1024×500 PNG (this asset carries alpha and was accepted as-is)
- `phoneScreenshots/` — 6 per locale (1080×2340 / 540×1170; the 20:9 ratio was accepted)
- `tenInchScreenshots/` — 6 per locale (1600×2560 tablet)
- `sevenInchScreenshots/` — empty (optional)

Filenames are zero-padded (`01`, `02`, …) so fastlane's alphabetical upload
order matches intent. Locale is inferred from the folder, not the filename.

## Lanes

All listing lanes require `version_code:` + `track:` (see above).

| Lane        | What it does |
| ----------- | ------------ |
| `tracks`    | List Play tracks + their releases/version codes (read-only) |
| `validate`  | Dry-run listing text + images + changelogs against the Play API (no changes) |
| `metadata`  | Upload listing text + images (no binary, no changelogs) |
| `changelog` | Upload only release notes |
| `pull`      | Pull the current live listing back into `fastlane/metadata/` |

## Usage

```bash
# one-time: place the Play Console service-account JSON (gitignored)
cp ~/Downloads/play-store-key.json fastlane/play-store-key.json

fastlane tracks                                            # list tracks + version codes
fastlane validate version_code:4 track:alpha               # dry-run, no changes
fastlane metadata version_code:4 track:alpha               # push listing text + images
fastlane changelog version_code:4 track:alpha              # release notes only
fastlane pull                                              # pull live listing down
```

- **`track`** for closed testing is the track ID (`alpha` / `beta` / your custom
  closed-track name), not `production`. Pass `track:` per command or set
  `ANDROID_TRACK`. Defaults to `production`.
- **`version_code`** is the integer Android version code of an existing release
  (EAS auto-increments it). Pass `version_code:` or set `ANDROID_VERSION_CODE`.
  Use `fastlane tracks` to find current values.

## Prerequisites

1. A Play Console **service account** with the Google Play Android Developer API
   enabled; download its JSON and save as `fastlane/play-store-key.json`
   (gitignored).
2. `fastlane` installed (`brew install fastlane` or `gem install fastlane`).
