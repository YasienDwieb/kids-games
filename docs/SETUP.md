# Setup & Build Guide

How to set up the project, test it on your phone with Expo Go, and build it
both locally and in the cloud with EAS.

Kids Games is **cross-platform** (Android, iOS, and web). You don't need to set
up every platform — pick the ones you care about and follow only those sections.
Android works from Linux/Windows/macOS; iOS builds require macOS (or EAS cloud).

---

## 1. Prerequisites

- **Node.js 18+** and npm
- **Git**
- The **Expo Go** app on your phone (for the fastest testing loop) —
  [Android (Play Store)](https://play.google.com/store/apps/details?id=host.exp.exponent) ·
  [iOS (App Store)](https://apps.apple.com/app/expo-go/id982107779)

Optional, only if you build native binaries locally:

- **Android:** Android Studio + an SDK and an emulator or a USB-connected device
- **iOS:** macOS with Xcode (+ CocoaPods)

You do **not** need a paid Apple/Google account just to develop and test.

---

## 2. Project setup

```bash
git clone <your-fork-url>
cd kids-games
npm install            # if peer-dep conflicts appear, use: npm install --legacy-peer-deps
```

Start the Metro dev server:

```bash
npm start              # or: npx expo start
```

This prints a QR code and a dev menu. From here you can run on a device, an
emulator/simulator, or the web.

---

## 3. Test on your phone with Expo Go

This is the quickest way to see changes — no native build required.

1. Run `npm start`.
2. Make sure your phone and computer are on the **same Wi-Fi network**.
3. Scan the QR code:
   - **Android:** scan it from inside the Expo Go app.
   - **iOS:** scan it with the Camera app, then open in Expo Go.
4. The app loads and **hot-reloads** as you edit code. Shake the device for the
   dev menu.

> On a restricted/office network the LAN connection can fail. Use a tunnel
> instead: `npx expo start --tunnel`.

---

## 4. Run on an emulator, simulator, or web

With the dev server running, press the matching key, or use the scripts:

```bash
npm run android        # Android emulator / connected device
npm run ios            # iOS simulator (macOS only)
npm run web            # open in a browser
```

Each platform is independent — running Android does not require iOS to be set
up, and vice versa.

---

## 5. Build locally

Local builds produce an installable binary on your own machine. They take longer
than Expo Go but let you test a real standalone app.

### Android (Linux / Windows / macOS)

Requires Android Studio + SDK.

```bash
# Debug build installed on a running emulator/device:
npx expo run:android

# Release APK/AAB via Gradle:
cd android
./gradlew assembleRelease     # APK  → android/app/build/outputs/apk/release/
./gradlew bundleRelease       # AAB  → android/app/build/outputs/bundle/release/
```

### iOS (macOS only)

Requires Xcode + CocoaPods.

```bash
npx expo run:ios              # debug build on the iOS simulator
```

For a signed release build, open `ios/` in Xcode and archive, or use EAS (below).

> The `android/` (and, after `expo run:ios`, the `ios/`) native folders are
> generated by Expo. If you change `app.json` native config, regenerate them with
> `npx expo prebuild`.

## 6. Build in the cloud with EAS

[EAS Build](https://docs.expo.dev/build/introduction/) compiles your app on
Expo's servers — handy when you don't have the native toolchain locally (e.g.
building iOS without a Mac).

### One-time setup

```bash
npm install -g eas-cli
eas login                     # free Expo account
eas build:configure           # creates eas.json with build profiles
```

### Build

```bash
eas build --platform android  # Android
eas build --platform ios      # iOS (needs Apple credentials; EAS can manage them)
eas build --platform all      # both
```

EAS prints a URL where you can watch progress and download the artifact when it
finishes. You can build for one platform without configuring the other.

### Build with the EAS pipeline, but on your own machine

Add `--local` to run the *same* EAS build process locally instead of in the
cloud — it uses your `eas.json` profiles but produces the artifact on your
machine (no queue, no build credits, no upload). Useful for debugging cloud
build failures or when builds must stay local.

```bash
eas build --local --platform android                       # debug profile
eas build --local --platform android --profile production  # production APK/AAB
```

This still requires the native toolchain locally (Android SDK). On Linux/Windows
only Android is supported with `--local`; iOS local builds need macOS — same as
the Gradle/Xcode builds in section 5. Unlike `expo run:android` (a plain debug
build) or raw Gradle, `--local` faithfully replicates the cloud pipeline.

### Submit to the stores (optional)

```bash
eas submit --platform android
eas submit --platform ios
```

## 7. Quality checks

Run these before pushing:

```bash
npx tsc --noEmit                 # TypeScript strict typecheck
npm test                         # Jest unit tests
npx expo export --platform ios   # verify the JS bundle resolves
```

## Troubleshooting

| Symptom | Fix |
|---|---|
| `npm install` fails on peer deps | `npm install --legacy-peer-deps` |
| Expo Go can't connect | Same Wi-Fi, or `npx expo start --tunnel` |
| Stale cache / weird Metro errors | `npx expo start -c` |
| Native config changes not applied | `npx expo prebuild` then rebuild |
| Wrong Expo dependency versions | `npx expo install <pkg>` instead of `npm install <pkg>` |
